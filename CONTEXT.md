# WINTR Context

Shared language for WINTR — the product and the project.

## Core Terms

**WINTR** — The product name. WIN Through Reasoning.

**Trifecta** — The three dimensions: UX, Business, Code. Experts in one miss the details that cause failure.

**Vibe coding** — Using AI to write code by prompting and tweaking. The naive approach many indie-hackers use. WINTR promises to do it better.

## Buyer Profiles

1. **Technical PM** — Primary buyer. Delivers products that matter — for employers, clients, or side-hustles. Wants to be top 1%.
2. **Indie-hacker** — Secondary buyer. Building side-hustles with AI. Wants to avoid failure.

Both share the same fear: wasting effort on something that was never going to work.

## Problem (Disease)

Jumping in too quickly and risking it all — building something doomed to fail because of gaps in expertise.
