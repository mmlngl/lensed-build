# Make Deck

Build a set of slides based on the user's input.

## Deck Structure

Every deck has three invisible sections separated by Chapter Title slides:

1. **Beginning** – The setup. Explains why the audience should pay attention and what they'll gain.
2. **Middle** – The main points, with tension: ups and downs, twists and turns.
3. **End** – Summary and clear next steps.

## Slide Types

- **Title slides** — Anchor each section. Bold title + (optional) one supporting line.
- **Content slides** — Headline + points + (optional) diagram suggestion.

## Transitions

Link slides with **BUT** or **THEREFORE** only — never "and", "so", "then", or any other connector.

- **BUT** — introduces contrast, complication, or a new angle
- **THEREFORE** — introduces a consequence, solution, or conclusion

## Workflow

1. Ask the user about the presentation goal, audience, and key message
2. If the core value is unclear, ask "why?" until it's sharp
3. Build the deck following the format below
4. Ask the user to review and iterate

## Output Format

### Title Slide

```
----
# Section Title

Supporting line
----
```

### Content Slide

```
----
[but/therefore]
[Slide N]

Headline text

Optional lead line

- point 1
- point 2
- diagram suggestion (optional)
----
```

### Section Layout

Structure the deck in three sections with title slides:

```
# Beginning
[Title slide]
[Content slides...]

# Middle
[Title slide]
[Content slides...]

# End
[Title slide]
[Content slides...]
```

## Safe Assumptions

The audience will be:

- Uninterested when the presentation begins
- Have little or no prior knowledge of the topic

Keep decks short. If it can't be condensed, it's not ready for a deck.
